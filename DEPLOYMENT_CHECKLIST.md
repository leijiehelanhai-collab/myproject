# 🚀 BSC主网部署清单

## ✅ 部署前准备

- [ ] 已创建 `.env` 文件
- [ ] 已填写 `PRIVATE_KEY`（部署账户私钥）
- [ ] 已填写 `DEV_WALLET`（开发者钱包地址）

### 代码检查
- [ ] 所有测试通过

### 安全审计（强烈推荐）
- [ ] 已使用 Slither 进行静态分析
- [ ] 已使用 Mythril 进行安全检查
- [ ] 考虑专业审计（CertiK、PeckShield等）

## 📋 部署步骤

### 1. 编译合约
```bash
npx hardhat compile
```
- [ ] 编译成功
- [ ] 无警告和错误

### 2. 执行部署
```bash
npx hardhat run scripts/deploy-mainnet.js --network bscMainnet
```
- [ ] 部署成功
- [ ] 记录合约地址
- [ ] 记录部署区块号
- [ ] 记录部署交易哈希

### 3. 验证合约
```bash
npx hardhat verify --network bscMainnet <合约地址> <开发者钱包>
```
- [ ] 验证成功
- [ ] 在BSCScan可见源代码

### 4. 更新配置
- [ ] 更新 `src/config.js` 中的合约地址
- [ ] 更新 `.env` 中的 `CONTRACT_ADDRESS`
- [ ] 确认前端网络配置（Chain ID: 56）

### 5. 前端构建
```bash
npm run build
```
- [ ] 构建成功
- [ ] 测试build文件夹

### 6. 前端部署
选择一种方式：
- [ ] 方式1：Netlify/Vercel
- [ ] 方式2：IPFS（去中心化）
- [ ] 方式3：自己的服务器
